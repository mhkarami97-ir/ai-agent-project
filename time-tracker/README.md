# Time Tracker

این پروژه یک برنامه تک‌صفحه‌ای کاملاً سمت کاربر برای ثبت و پیگیری زمان کارها طراحی شده است که می‌توان آن را روی GitHub Pages منتشر کرد.

## ساختار پروژه
- `index.html` – فایل اصلی HTML با جهت‌گیری راست به چپ، فرم افزودن کار جدید، فهرست کارها، و بخش گزارش روزانه.
- `assets/css/styles.css` – تمام سبک‌ها در این فایل قرار گرفته، شامل سربرگ، کارت‌ها، دکمه‌ها، و واکنش‌گرایی برای موبایل.
- `assets/js/app.js` – منطق کار با DOM، ایجاد/ویرایش/حذف وظایف، شروع/توقف زمان، محاسبه گزارش روزانه و ذخیره‌سازی در `localStorage`.

## راه‌اندازی و اجرا
1. مخزن را کلون کنید یا فایل‌ها را در پوشه جدید قرار دهید.
2. کافی است `index.html` را در مرورگر باز کنید یا مخزن را روی GitHub Pages قرار دهید.
   - برای انتشار: پوشه را در شاخه `gh-pages` یا `main` (با تنظیمات Pages) قرار دهید.
3. نیازی به سرور یا پیکربندی اضافی نیست؛ همه اطلاعات در مرورگر ذخیره می‌شوند.

## استفاده
- فرم بالای صفحه را پر کنید تا کار جدیدی با عنوان و دسته‌بندی ثبت کنید.
- برای هر کار دکمه «شروع»/«توقف» را بزنید تا زمان ثبت شود؛ وضعیت در همان کارت نمایش داده می‌شود.
- با دکمه «ویرایش» عنوان و دسته‌بندی را تغییر دهید یا با «حذف» آن را پاک کنید.
- گزارش روزانه مجموع زمان‌های صرف‌شده را بر اساس تاریخ نشان می‌دهد.

## توسعه
- فقط از HTML/CSS/JS استفاده شده و وابستگی خارجی خاصی جز فونت و پلتفرم مرورگر وجود ندارد.
- برای تغییر قالب یا افزودن ویژگی‌های جدید کافی است فایل‌های داخل `assets` را ویرایش کنید.
- از `localStorage` برای نگهداری داده‌ها استفاده می‌شود تا بدون هیچ بک‌اندی داده‌ها حفظ شوند.

## تست
- صفحه را در مرورگر باز کنید و با ثبت چند کار مختلف وضعیت کارت‌ها و گزارش را بررسی کنید.
- روی دستگاه موبایل یا با ابزارهای توسعه‌دهنده مرورگر جدول‌بندی واکنش‌پذیری را تست کنید.
