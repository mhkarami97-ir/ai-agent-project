# راهنمای استقرار در GitHub Pages

این راهنما به شما کمک می‌کند تا پروژه خود را در GitHub Pages مستقر کنید.

## مراحل استقرار

### 1. ساخت مخزن در GitHub

1. به [GitHub](https://github.com) بروید
2. روی دکمه **New repository** کلیک کنید
3. نام مخزن را وارد کنید (مثلاً `text-counter`)
4. گزینه **Public** را انتخاب کنید
5. روی **Create repository** کلیک کنید

### 2. آپلود فایل‌ها به GitHub

#### روش اول: استفاده از Git (توصیه می‌شود)

```bash
# در پوشه پروژه، ترمینال را باز کنید و دستورات زیر را اجرا کنید:

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git push -u origin main
```

**نکته**: `USERNAME` را با نام کاربری GitHub خود و `REPOSITORY` را با نام مخزن جایگزین کنید.

#### روش دوم: آپلود مستقیم از وب

1. وارد مخزن خود در GitHub شوید
2. روی **Add file** و سپس **Upload files** کلیک کنید
3. تمام فایل‌های پروژه را بکشید و رها کنید
4. روی **Commit changes** کلیک کنید

### 3. فعال‌سازی GitHub Pages

#### روش خودکار (با GitHub Actions):
فایل `.github/workflows/deploy.yml` به صورت خودکار پروژه را دیپلوی می‌کند.

1. به تنظیمات مخزن بروید: **Settings** > **Pages**
2. در قسمت **Source** گزینه **GitHub Actions** را انتخاب کنید
3. فایل‌ها را push کنید و منتظر بمانید تا Actions اجرا شود
4. پس از اتمام، لینک سایت شما نمایش داده می‌شود

#### روش دستی:

1. به تنظیمات مخزن بروید: **Settings** > **Pages**
2. در قسمت **Source** گزینه **Deploy from a branch** را انتخاب کنید
3. **Branch** را روی `main` و پوشه را روی `/ (root)` تنظیم کنید
4. روی **Save** کلیک کنید
5. چند دقیقه صبر کنید تا سایت شما آماده شود

### 4. مشاهده سایت

پس از فعال‌سازی موفقیت‌آمیز، سایت شما در آدرس زیر در دسترس خواهد بود:

```
https://USERNAME.github.io/REPOSITORY/
```

مثال:
```
https://myusername.github.io/text-counter/
```

## نکات مهم

### نام دامنه شخصی

اگر می‌خواهید از دامنه شخصی استفاده کنید:

1. به **Settings** > **Pages** بروید
2. در قسمت **Custom domain** دامنه خود را وارد کنید
3. تنظیمات DNS دامنه خود را مطابق راهنمای GitHub پیکربندی کنید

### HTTPS

GitHub Pages به صورت خودکار از HTTPS پشتیبانی می‌کند. مطمئن شوید که گزینه **Enforce HTTPS** فعال است.

### به‌روزرسانی سایت

هر بار که تغییراتی ایجاد کنید و آن‌ها را push کنید، سایت به صورت خودکار به‌روزرسانی می‌شود.

## عیب‌یابی

### سایت نمایش داده نمی‌شود

1. مطمئن شوید که GitHub Pages فعال است
2. بررسی کنید که فایل `index.html` در ریشه پروژه قرار دارد
3. صبر کنید تا فرآیند build کامل شود (ممکن است چند دقیقه طول بکشد)

### تغییرات نمایش داده نمی‌شوند

1. کش مرورگر را پاک کنید (Ctrl+Shift+R)
2. چند دقیقه صبر کنید تا GitHub Pages به‌روزرسانی شود
3. بررسی کنید که تغییرات به درستی push شده باشند

### خطای 404

1. مطمئن شوید که به آدرس صحیح مراجعه می‌کنید
2. بررسی کنید که نام فایل‌ها درست باشد (حساس به حروف بزرگ و کوچک)
