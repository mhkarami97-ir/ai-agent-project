# راهنمای استقرار در GitHub Pages

این راهنما شما را گام به گام در استقرار پروژه روی GitHub Pages همراهی می‌کند.

## پیش‌نیازها

- حساب کاربری GitHub
- نصب Git روی سیستم شما

## مراحل استقرار

### مرحله 1: ایجاد مخزن در GitHub

1. به [GitHub](https://github.com) بروید و وارد حساب کاربری خود شوید
2. روی دکمه "+" در گوشه بالا سمت راست کلیک کنید
3. گزینه "New repository" را انتخاب کنید
4. نام مخزن را وارد کنید (مثلاً: `text-formatter`)
5. توضیحات (اختیاری): "ابزار پاک‌سازی متن فارسی و انگلیسی"
6. مخزن را Public نگه دارید (برای GitHub Pages رایگان)
7. روی "Create repository" کلیک کنید

### مرحله 2: آپلود کردن فایل‌ها

دو روش برای آپلود فایل‌ها وجود دارد:

#### روش A: استفاده از Git (پیشنهادی)

1. در پوشه پروژه، ترمینال یا Command Prompt را باز کنید
2. دستورات زیر را به ترتیب اجرا کنید:

```bash
# مقداردهی اولیه Git
git init

# اضافه کردن تمام فایل‌ها
git add .

# ایجاد اولین commit
git commit -m "Initial commit: Persian text formatter app"

# تنظیم نام branch به main
git branch -M main

# اضافه کردن آدرس مخزن GitHub (آدرس مخزن خود را جایگزین کنید)
git remote add origin https://github.com/USERNAME/text-formatter.git

# ارسال فایل‌ها به GitHub
git push -u origin main
```

**نکته**: `USERNAME` را با نام کاربری GitHub خود جایگزین کنید.

#### روش B: آپلود مستقیم از رابط GitHub

1. در صفحه مخزن خود، روی "uploading an existing file" کلیک کنید
2. تمام فایل‌های پروژه را انتخاب و آپلود کنید
3. پیام commit را وارد کرده و روی "Commit changes" کلیک کنید

### مرحله 3: فعال‌سازی GitHub Pages

1. در صفحه مخزن، روی تب **Settings** کلیک کنید
2. در منوی سمت چپ، روی **Pages** کلیک کنید
3. در بخش **Source**:
   - Branch: `main` را انتخاب کنید
   - Folder: `/ (root)` را انتخاب کنید
4. روی دکمه **Save** کلیک کنید

### مرحله 4: دسترسی به سایت

بعد از چند دقیقه، سایت شما در آدرس زیر منتشر می‌شود:

```
https://USERNAME.github.io/text-formatter/
```

**نکته**: `USERNAME` را با نام کاربری GitHub خود جایگزین کنید.

## به‌روزرسانی سایت

هر زمان که می‌خواهید سایت را به‌روزرسانی کنید:

```bash
git add .
git commit -m "توضیح تغییرات"
git push
```

بعد از چند دقیقه، تغییرات شما روی سایت اعمال می‌شود.

## حل مشکلات رایج

### مشکل 404 (صفحه پیدا نشد)

- مطمئن شوید که فایل `index.html` در ریشه مخزن قرار دارد
- صبر کنید 5-10 دقیقه تا GitHub Pages به‌روزرسانی شود
- کش مرورگر را پاک کنید (Ctrl+Shift+Delete)

### فونت‌ها درست نمایش داده نمی‌شوند

- مطمئن شوید که اتصال اینترنت برقرار است (فونت از CDN بارگذاری می‌شود)
- کش مرورگر را پاک کنید

### استایل‌ها اعمال نمی‌شوند

- مسیر فایل CSS را در `index.html` بررسی کنید
- مطمئن شوید که ساختار پوشه‌ها درست است:
  ```
  text-formatter/
  ├── index.html
  └── assets/
      ├── css/
      │   └── style.css
      └── js/
          └── script.js
  ```

## تنظیمات اضافی (اختیاری)

### اضافه کردن دامنه سفارشی

1. در تنظیمات GitHub Pages، بخش "Custom domain" را پیدا کنید
2. دامنه خود را وارد کنید (مثلاً: `textformatter.com`)
3. در تنظیمات DNS دامنه خود، یک رکورد CNAME به `USERNAME.github.io` اضافه کنید

### فعال‌سازی HTTPS

GitHub Pages به صورت خودکار HTTPS را فعال می‌کند. اگر فعال نیست:

1. در تنظیمات Pages، گزینه "Enforce HTTPS" را فعال کنید
2. صبر کنید تا گواهی SSL صادر شود (ممکن است چند ساعت طول بکشد)
