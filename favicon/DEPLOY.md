# 🚀 راهنمای استقرار در GitHub Pages

## مراحل استقرار

### 1. ایجاد مخزن GitHub

1. وارد [GitHub](https://github.com) شوید
2. روی دکمه **New repository** کلیک کنید
3. نام مخزن را وارد کنید (مثلاً `icon-generator`)
4. مخزن را **Public** قرار دهید
5. روی **Create repository** کلیک کنید

### 2. آپلود فایل‌ها

#### روش اول: از طریق وب (ساده)

1. در صفحه مخزن، روی **uploading an existing file** کلیک کنید
2. تمام فایل‌های پروژه را بکشید و در صفحه رها کنید
3. پیام commit را بنویسید و روی **Commit changes** کلیک کنید

#### روش دوم: از طریق Git (پیشرفته)

```bash
# ابتدا git را راه‌اندازی کنید
git init

# فایل‌ها را اضافه کنید
git add .

# یک commit ایجاد کنید
git commit -m "Initial commit"

# شاخه اصلی را main بنامید
git branch -M main

# مخزن GitHub را اضافه کنید (URL خود را جایگزین کنید)
git remote add origin https://github.com/username/icon-generator.git

# فایل‌ها را push کنید
git push -u origin main
```

### 3. فعال‌سازی GitHub Pages

1. وارد تنظیمات مخزن شوید (**Settings**)
2. از منوی سمت چپ، روی **Pages** کلیک کنید
3. در بخش **Source**:
   - Branch: `main` را انتخاب کنید
   - Folder: `/ (root)` را انتخاب کنید
4. روی **Save** کلیک کنید
5. چند دقیقه صبر کنید تا سایت منتشر شود

### 4. دسترسی به سایت

سایت شما در آدرس زیر در دسترس خواهد بود:

```
https://username.github.io/icon-generator/
```

(username و icon-generator را با نام کاربری و نام مخزن خود جایگزین کنید)

## 🎨 دامنه سفارشی (اختیاری)

اگر می‌خواهید از دامنه شخصی خود استفاده کنید:

1. یک فایل `CNAME` در روت پروژه ایجاد کنید
2. نام دامنه خود را در آن بنویسید (مثلاً `icongen.example.com`)
3. در تنظیمات DNS دامنه خود، یک رکورد CNAME اضافه کنید که به `username.github.io` اشاره کند

## ✅ بررسی و تست

پس از انتشار:

1. به آدرس GitHub Pages خود بروید
2. عملکرد تمام قسمت‌های سایت را بررسی کنید
3. سایت را در موبایل و مرورگرهای مختلف تست کنید

## 🔄 به‌روزرسانی

برای به‌روزرسانی سایت:

1. تغییرات را در فایل‌های محلی اعمال کنید
2. فایل‌های جدید را commit و push کنید
3. GitHub به طور خودکار سایت را به‌روزرسانی می‌کند (ممکن است چند دقیقه طول بکشد)

```bash
git add .
git commit -m "توضیح تغییرات"
git push
```

## 🐛 رفع مشکلات رایج

### سایت نمایش داده نمی‌شود

- چند دقیقه صبر کنید (اولین استقرار ممکن است ۱۰ دقیقه طول بکشد)
- بررسی کنید که GitHub Pages در تنظیمات فعال باشد
- اطمینان حاصل کنید که فایل `index.html` در روت مخزن قرار دارد

### CSS یا JavaScript کار نمی‌کند

- مسیرهای فایل‌ها را بررسی کنید (باید نسبی باشند، نه مطلق)
- کنسول مرورگر را برای خطاها بررسی کنید (F12)

### تصاویر نمایش داده نمی‌شوند

- از مسیرهای نسبی استفاده کنید (مثلاً `./assets/image.png` به جای `/assets/image.png`)

## 📊 آمار و تحلیل

برای افزودن Google Analytics یا سایر ابزارهای آماری:

1. کد ردیابی را دریافت کنید
2. آن را قبل از تگ `</head>` در فایل `index.html` قرار دهید

## 🔒 امنیت

- هیچ‌گاه اطلاعات حساس را در مخزن Public قرار ندهید
- از HTTPS استفاده کنید (GitHub Pages به طور پیش‌فرض از HTTPS پشتیبانی می‌کند)

## 💡 نکات مفید

- می‌توانید Actions را برای خودکارسازی استقرار استفاده کنید
- برای بهبود سئو، فایل `sitemap.xml` اضافه کنید
- برای بهبود عملکرد، تصاویر را بهینه‌سازی کنید
