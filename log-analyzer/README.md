# 📊 تحلیلگر لاگ محلی

یک ابزار تحلیل لاگ قدرتمند که کاملاً در مرورگر شما اجرا می‌شود. بدون نیاز به سرور یا ارسال داده‌ها به اینترنت.

## ✨ ویژگی‌ها

### 🎯 امکانات اصلی
- **بارگذاری فایل‌های لاگ**: پشتیبانی از فرمت‌های TXT، JSON و LOG
- **جستجوی متنی**: جستجوی سریع در محتوای لاگ‌ها
- **فیلتر پیشرفته**: فیلتر بر اساس سطح لاگ (ERROR، WARN، INFO، DEBUG)
- **جستجوی Regex**: استفاده از الگوهای Regular Expression برای یافتن الگوهای پیچیده
- **گروه‌بندی هوشمند**: گروه‌بندی لاگ‌ها بر اساس سطح، تاریخ یا منبع
- **Syntax Highlighting**: رنگ‌آمیزی خودکار بر اساس سطح لاگ
- **آمار زنده**: نمایش آمار و تعداد انواع لاگ‌ها
- **خروجی JSON**: امکان دریافت خروجی تحلیل به صورت JSON
- **ذخیره‌سازی محلی**: ذخیره تحلیل‌ها در IndexedDB مرورگر
- **رابط کاربری فارسی**: طراحی کاملاً فارسی و RTL

### 🎨 طراحی
- **رسپانسیو**: سازگار با تمام اندازه‌های صفحه (موبایل، تبلت، دسکتاپ)
- **مدرن و زیبا**: استفاده از Gradient ها و انیمیشن‌های نرم
- **فونت Vazirmatn**: استفاده از فونت مدرن فارسی وزیرمتن
- **تم تیره**: محیط نمایش لاگ با تم تیره برای راحتی بیشتر چشم

### 💾 ذخیره‌سازی
- استفاده از IndexedDB مرورگر برای ذخیره تحلیل‌ها
- امکان ذخیره چندین تحلیل مختلف
- بارگذاری سریع تحلیل‌های قبلی
- مدیریت تحلیل‌های ذخیره شده (مشاهده، بارگذاری، حذف)

## 🚀 نحوه استفاده

### راه‌اندازی محلی

1. فایل‌های پروژه را دانلود کنید
2. فایل `index.html` را در مرورگر باز کنید
3. شروع به استفاده کنید!

### راه‌اندازی در GitHub Pages

1. یک repository جدید در GitHub بسازید
2. تمام فایل‌های پروژه را به repository آپلود کنید
3. به Settings > Pages بروید
4. در قسمت Source، branch را انتخاب کنید (معمولاً main یا master)
5. صفحه شما در آدرس `https://username.github.io/repository-name` در دسترس خواهد بود

## 🔧 نحوه کار با برنامه

### 1. بارگذاری فایل لاگ
- روی ناحیه بارگذاری کلیک کنید یا فایل را به آن بکشید
- فایل‌های با فرمت TXT، JSON یا LOG پشتیبانی می‌شوند

### 2. مشاهده آمار
- بعد از بارگذاری، آمار کلی لاگ‌ها نمایش داده می‌شود
- تعداد کل خطوط، خطاها، هشدارها و اطلاعات

### 3. جستجو و فیلتر
- **جستجوی متنی**: متن مورد نظر را در کادر جستجو وارد کنید
- **فیلتر سطح**: سطح لاگ مورد نظر را انتخاب کنید (ERROR، WARN، INFO، DEBUG)
- **Regex**: الگوهای پیچیده‌تر را با Regex جستجو کنید (مثال: `ERROR|FATAL`)

### 4. گروه‌بندی
- گزینه گروه‌بندی را انتخاب کنید:
  - **بدون گروه‌بندی**: نمایش معمولی
  - **سطح لاگ**: گروه‌بندی بر اساس ERROR، WARN، INFO، DEBUG
  - **تاریخ**: گروه‌بندی بر اساس تاریخ
  - **منبع**: گروه‌بندی بر اساس منبع لاگ

### 5. خروجی و ذخیره
- **خروجی JSON**: دریافت فایل JSON با تمام اطلاعات تحلیل
- **ذخیره تحلیل**: ذخیره تحلیل فعلی در مرورگر
- **بارگذاری تحلیل**: بارگذاری تحلیل‌های قبلی ذخیره شده

### 6. ابزارهای نمایش
- **کپی**: کپی کردن محتوای نمایش داده شده
- **تغییر حالت شکستن خط**: تغییر بین حالت wrap و no-wrap

## 🎨 ویژگی‌های طراحی

### رنگ‌بندی
- **ERROR**: قرمز
- **WARN**: نارنجی
- **INFO**: آبی
- **DEBUG**: خاکستری

### رسپانسیو
- **دسکتاپ**: نمایش کامل تمام امکانات
- **تبلت**: بهینه‌سازی برای صفحه‌های متوسط
- **موبایل**: چیدمان عمودی و دکمه‌های بزرگ‌تر

## 🛠 تکنولوژی‌ها

- **HTML5**: ساختار صفحه
- **CSS3**: طراحی و استایل
- **Vanilla JavaScript**: منطق برنامه (بدون وابستگی به کتابخانه‌های خارجی)
- **IndexedDB**: ذخیره‌سازی محلی
- **File API**: خواندن فایل‌ها
- **Regex**: جستجوی پیشرفته
- **Vazirmatn Font**: فونت فارسی مدرن

## 📝 نکات مهم

### امنیت
- تمام عملیات در مرورگر شما انجام می‌شود
- هیچ داده‌ای به سرور ارسال نمی‌شود
- فایل‌های لاگ شما کاملاً خصوصی می‌مانند

### محدودیت‌ها
- اندازه فایل بستگی به حافظه مرورگر دارد (معمولاً تا چند صد مگابایت)
- تحلیل‌های ذخیره شده فقط در همان مرورگر و دستگاه قابل دسترس هستند
- پاک کردن cache مرورگر، تحلیل‌های ذخیره شده را حذف می‌کند

### بهترین شیوه‌ها
- برای فایل‌های بزرگ، از فیلترها برای محدود کردن نتایج استفاده کنید
- از Regex برای پیدا کردن الگوهای پیچیده استفاده کنید
- تحلیل‌های مهم را به صورت JSON خروجی بگیرید و ذخیره کنید
