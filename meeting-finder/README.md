# یافتن زمان مناسب جلسه (Meeting Time Finder)

یک برنامه تحت وب برای پیدا کردن بهترین زمان مشترک برای برگزاری جلسه بین چند نفر.

## 📋 توضیحات

این برنامه به شما کمک می‌کند تا با وارد کردن بازه‌های زمانی آزاد هر یک از شرکت‌کنندگان، زمان‌های مشترکی که همه افراد در آن زمان‌ها آزاد هستند را پیدا کنید. این ابزار برای هماهنگی جلسات تیمی، قرار ملاقات‌ها و هر نوع برنامه‌ریزی گروهی بسیار مفید است.

## ✨ ویژگی‌ها

- ✅ **رابط کاربری فارسی و RTL**: کاملاً به زبان فارسی و با پشتیبانی از راست‌چین
- 📱 **طراحی ریسپانسیو**: قابل استفاده در موبایل، تبلت و دسکتاپ
- 👥 **مدیریت شرکت‌کنندگان**: افزودن، حذف و ویرایش شرکت‌کنندگان
- ⏰ **مدیریت بازه‌های زمانی**: تعیین چندین بازه زمانی برای هر شرکت‌کننده
- 🔍 **یافتن زمان‌های مشترک**: الگوریتم هوشمند برای پیدا کردن همه بازه‌های زمانی مشترک
- 💾 **ذخیره‌سازی محلی**: تمام اطلاعات به صورت محلی در مرورگر ذخیره می‌شود
- 🎨 **طراحی مدرن**: رابط کاربری زیبا با رنگ‌های گرادیانت و انیمیشن‌ها

## 🚀 نحوه استفاده

### 1. افزودن شرکت‌کنندگان

1. نام شرکت‌کننده را در فیلد "نام شرکت‌کننده" وارد کنید
2. روی دکمه "افزودن شرکت‌کننده" کلیک کنید
3. این کار را برای تمام افرادی که می‌خواهید در جلسه شرکت کنند، تکرار کنید

### 2. تعیین بازه‌های زمانی

1. روی دکمه "ویرایش" کنار نام هر شرکت‌کننده کلیک کنید
2. تاریخ و ساعت شروع بازه زمانی آزاد را وارد کنید
3. تاریخ و ساعت پایان را وارد کنید
4. روی "افزودن بازه زمانی" کلیک کنید
5. می‌توانید چندین بازه زمانی برای هر شخص اضافه کنید
6. پس از اتمام، روی "بازگشت" کلیک کنید

### 3. یافتن زمان مشترک

1. پس از اضافه کردن بازه‌های زمانی برای حداقل دو نفر
2. روی دکمه "یافتن زمان‌های مشترک" کلیک کنید
3. برنامه تمام بازه‌های زمانی که همه افراد در آن‌ها آزاد هستند را نمایش می‌دهد

## 🛠️ تکنولوژی‌های استفاده شده

- **HTML5**: ساختار صفحه
- **CSS3**: استایل‌دهی و طراحی ریسپانسیو
- **JavaScript (Vanilla)**: منطق برنامه و الگوریتم پیدا کردن همپوشانی‌ها
- **LocalStorage API**: ذخیره‌سازی داده‌ها در مرورگر

## 💻 نصب و اجرا

### روش 1: اجرا به صورت محلی

1. فایل‌های پروژه را دانلود کنید
2. فایل `index.html` را با مرورگر خود باز کنید
3. شروع به استفاده کنید!

### روش 2: میزبانی روی GitHub Pages

1. یک مخزن (Repository) جدید در GitHub ایجاد کنید
2. تمام فایل‌های پروژه را به مخزن آپلود کنید:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPOSITORY.git
   git push -u origin main
   ```
3. به تنظیمات مخزن بروید (Settings)
4. در بخش Pages، منبع را روی `main branch` تنظیم کنید
5. پس از چند دقیقه، سایت شما در آدرس `https://USERNAME.github.io/REPOSITORY` در دسترس خواهد بود

- هر شرکت‌کننده حداقل یک بازه زمانی دارد
- بازه‌های زمانی با یکدیگر همپوشانی دارند

---

**سوال**: چگونه می‌توانم داده‌ها را پاک کنم؟

## 📝 الگوریتم پیدا کردن همپوشانی

برنامه از الگوریتم Sweep Line استفاده می‌کند که به این صورت کار می‌کند:

1. تمام بازه‌های زمانی همه شرکت‌کنندگان جمع‌آوری می‌شوند
2. نقاط شروع و پایان هر بازه به عنوان "رویداد" ثبت می‌شوند
3. رویدادها بر اساس زمان مرتب می‌شوند
4. با پیمایش رویدادها، زمان‌هایی که همه شرکت‌کنندگان فعال هستند شناسایی می‌شوند
5. بازه‌های مجاور یا همپوشان با یکدیگر ادغام می‌شوند
